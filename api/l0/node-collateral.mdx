---
title: "Node Collateral"
description: "Post and query node collateral positions on the Global L0 node."
---

Node collateral is a mechanism by which node operators lock DAG tokens as a security deposit to participate in consensus. Like delegated stakes, node collateral is backed by a `TokenLock` created on the DAG L1 layer.

<Note>
  All endpoints return the following response headers:
  - **X-Id** ã The node's public key identifier
  - **X-Session-Token** ã The current cluster session token
  - **Request-Signature** ã Cryptographic signature of the response for verification
</Note>

<Info>
  **Prerequisites:** Before posting node collateral, you must create a `TokenLock` via the DAG L1 `/token-locks` endpoint. The hash of that token lock transaction is used as `tokenLockRef` in collateral requests.
</Info>

## POST /node-collateral

Submits a signed node collateral creation request. The collateral will be included in the next global snapshot.

```bash
curl -X POST https://<node-host>:9000/node-collateral \
  -H "Content-Type: application/json" \
  -d '{
    "value": {
      "source": "DAG3hZTBgtb2iH9ZdLNvaKqPT4ZncCuW6uYxZack",
      "nodeId": "c7f9a08bdea7ff5f51c8af16e223a1d751bac9c541125d9aef5658e9b7597aee",
      "amount": 125000,
      "fee": 123,
      "tokenLockRef": "abc123...",
      "parent": { "ordinal": 0, "hash": "" }
    },
    "proofs": [
      { "id": "<public-key-hex>", "signature": "<ecdsa-der-hex>" }
    ]
  }'
```

**Request body** ã `SignedCreateNodeCollateral`

      DAG wallet address of the collateral provider
    </ParamField>
    <ParamField body="nodeId" type="string" required>
      Public key ID of the node being collateralized
    </ParamField>
    <ParamField body="amount" type="integer" required>
      Collateral amount in datums (must be ãË 0)
    </ParamField>
    <ParamField body="fee" type="integer">
      Optional transaction fee in datums (must be ãË 0)
    </ParamField>
    <ParamField body="tokenLockRef" type="string" required>
      Hash of the `TokenLock` transaction backing this collateral
    </ParamField>
    <ParamField body="parent" type="object" required>

      <Expandable title="NodeCollateralReference">

**Response `200 OK`**

## PUT /node-collateral

Withdraws an existing node collateral position. The locked tokens are released after the unlock period has elapsed.

```bash
curl -X PUT https://<node-host>:9000/node-collateral \
  -H "Content-Type: application/json" \
  -d '{
    "value": {
      "source": "DAG3hZTBgtb2iH9ZdLNvaKqPT4ZncCuW6uYxZack",
      "tokenLockRef": "abc123...",
      "parent": { "ordinal": 1, "hash": "abc123..." }
    },
    "proofs": [
      { "id": "<public-key-hex>", "signature": "<ecdsa-der-hex>" }
    ]
  }'
```

**Request body** ã `SignedWithdrawNodeCollateral`

Same envelope structure as `POST /node-collateral` but with a `WithdrawNodeCollateral` payload containing `source`, `tokenLockRef`, and `parent`.

**Response `200 OK`**

<ResponseField name="hash" type="string">
  Hash of the accepted node collateral withdrawal transaction
</ResponseField>

**Response `400 Bad Request`**

Invalid withdrawal request. Returns an `ErrorResponse` with rejection reasons.

**Response `503 Service Unavailable`**

Node is not in `Ready` state.

## GET /node-collateral/{address}/info

Returns all node collateral positions (active and pending withdrawal) for a given address.

```bash
curl https://<node-host>:9000/node-collateral/DAG3hZTBgtb2iH9ZdLNvaKqPT4ZncCuW6uYxZack/info
```

<ParamField path="address" type="string" required>
  DAG wallet address to query
</ParamField>

**Response `200 OK`**

<ResponseField name="address" type="string">
  The queried wallet address
</ResponseField>
<ResponseField name="activeNodeCollaterals" type="array">
  Currently active node collateral positions
  <Expandable title="NodeCollateralInfo">
    <ResponseField name="nodeId" type="string">
      The collateralized node's public key ID
    </ResponseField>
    <ResponseField name="acceptedOrdinal" type="integer">
      Snapshot ordinal when the collateral was accepted
    </ResponseField>
    <ResponseField name="tokenLockRef" type="string">
      Hash of the backing token lock transaction
    </ResponseField>
    <ResponseField name="amount" type="integer">
      Collateral amount in datums
    </ResponseField>
    <ResponseField name="fee" type="integer">
      Fee paid in datums
    </ResponseField>
    <ResponseField name="withdrawalStartEpoch" type="integer">
      Epoch when withdrawal was initiated (null if active)
    </ResponseField>
    <ResponseField name="withdrawalEndEpoch" type="integer">
      Epoch when withdrawal completes (null if active)
    </ResponseField>
  </Expandable>
</ResponseField>
<ResponseField name="pendingWithdrawals" type="array">
  Collateral positions in the withdrawal pending state (same schema as `activeNodeCollaterals`)
</ResponseField>

**Response `503 Service Unavailable`**

Node is not in `Ready` state.

## GET /node-collateral/last-reference/{address}

Returns the last node collateral transaction reference for a given address. Use this to build the `parent` field for new collateral requests.

```bash
curl https://<node-host>:9000/node-collateral/last-reference/DAG3hZTBgtb2iH9ZdLNvaKqPT4ZncCuW6uYxZack
```

<ParamField path="address" type="string" required>
  DAG wallet address
</ParamField>

**Response `200 OK`**

<ResponseField name="ordinal" type="integer">
  Ordinal of the last node collateral transaction (0 if none)
</ResponseField>
<ResponseField name="hash" type="string">
  Hash of the last node collateral transaction (empty string if none)
</ResponseField>

**Example response:**

```json
{
  "ordinal": 1,
  "hash": "a1b2c3d4e5f6..."
}
```

**Response `503 Service Unavailable`**

Node is not in `Ready` state.
