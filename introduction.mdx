Tessellation implements a hierarchical DAG (Directed Acyclic Graph) consensus protocol where **Layer 1 metagraphs** create blocks that **Layer 0** aggregates into global snapshots. This architecture enables scalable, parallel transaction processing across multiple metagraphs while maintaining a unified global state.

Written in **Scala 2.13** with **Cats Effect 3** for purely functional, resource-safe concurrency.

## What is Tessellation?

Tessellation is the node software powering the Constellation Network â€” a public DAG-based blockchain that supports parallel metagraph execution. Each metagraph runs its own L1 consensus, producing blocks that flow upward through Currency L0 and into the Global L0 for final global state commitment.

```
L1 Metagraph Nodes  â†’  Currency L0  â†’  Global L0  â†’  Global Snapshots
      (blocks)           (snapshots)    (consensus)     (final state)
```

The result is a network that can process transactions across many independent metagraphs simultaneously, with each metagraph able to carry custom data, reward schemes, and validation logic.

## Hierarchical architecture

The network is organized into three layers:

| Layer | Module | Role |
|-------|--------|------|
| Layer 1 â€” Metagraph | `dag-l1`, `currency-l1` | Block creation and per-metagraph consensus |
| Currency Layer 0 | `currency-l0` | Aggregates L1 blocks into currency snapshots |
| Global Layer 0 | `dag-l0` | Aggregates all currency snapshots into global snapshots |

**Consensus flow**: L1 creates blocks â†’ Currency-L0 creates snapshots â†’ Global-L0 creates global snapshots with all metagraph state.

## Key design patterns

Tessellation uses a consistent set of functional programming patterns across all modules:

- **Tagless final** â€” effect type `F[_]` throughout, with `IO` at the application boundary
- **Resource-based lifecycle** â€” `Resource[IO, A]` for all services and storages
- **Refined types** â€” `PosLong`, `NonNegLong`, and other refined wrappers for compile-time guarantees
- **Newtype wrappers** â€” zero-cost type safety for domain concepts
- **Monocle optics** â€” immutable data transformation via lenses
- **Droste recursion schemes** â€” hylomorphisms in the L0 event processing pipeline
- **Kleisli composition** â€” handler chains in the gossip and HTTP layers
